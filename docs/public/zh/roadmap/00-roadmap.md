# 路线图

本页描述 Aionis 对外可见的产品演进方向。

## 路线图原则

1. 记忆可验证与可回放是底座能力，不妥协。
2. 扩展策略闭环能力时保持可控性。
3. 通过显式预算持续优化上下文质量。
4. 每个能力扩展都同步运维就绪要求。

## 能力轨道

### 轨道 A：上下文智能

重点：

1. 面向场景的分层预设。
2. 更强的预算编排控制。
3. 合并/丢弃决策解释能力增强。

成功指标：

1. 固定 token 预算下回答质量提升。
2. 生产场景上下文拼装延迟稳定。

### 轨道 B：策略与执行治理

重点：

1. 更深的 planner-native 策略契约。
2. 更完整的决策溯源覆盖。
3. 更稳的策略演进与发布护栏。

成功指标：

1. 决策链路与反馈链路覆盖率提升。
2. 策略更新导致的回归下降。

### 轨道 C：可运营与证据化

重点：

1. 门禁自动化增强。
2. 基准与证据包能力增强。
3. 事故回放链路提速。

成功指标：

1. 故障演练中的定位时间下降。
2. 发布证据包完整度提升。

### 轨道 D：生态集成

重点：

1. MCP 与框架集成模式增强。
2. SDK 生产可用性与易用性提升。
3. 多运行时互操作指南完善。
4. 通过 Aionis Doc 接入 document-driven workflow。

成功指标：

1. 集成到首个可用价值时间缩短。
2. 集成特有故障率下降。

## 规划时间层

### 近期

1. 强化上下文编排预设与调优指南。
2. 标准化基准快照复现流程。
3. 增强策略闭环运维可观测性。
4. 把 Aionis Doc 正式化成公开的 executable-document workflow surface。

### 中期

1. 扩展 planner-native 策略契约。
2. 增强 URI 链路回放工具。
3. 完善混合运行时接入指南。

### 长期

1. 支持更长周期策略记忆能力。
2. 复杂编排图下的多层上下文编排。
3. 面向企业规模的记忆运营模式。

## 使用建议

1. 把路线图视为能力方向，不是固定发布时间承诺。
2. 按你的约束（延迟、可控、合规）择优优先级。
3. 升级前用生产门禁与基准证据验证。
