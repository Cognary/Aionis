# Roadmap

This roadmap describes publicly visible product evolution priorities for Aionis.

## Roadmap Principles

1. Keep memory verifiability and replayability as non-negotiable foundations.
2. Expand policy-loop capability without sacrificing controllability.
3. Improve context quality with explicit cost and latency controls.
4. Keep operational readiness coupled to every capability expansion.

## Capability Tracks

### Track A: Context Intelligence

Focus:

1. richer layer presets for domain workloads
2. stronger budget-aware composition controls
3. better explainability for merge/drop decisions

Success indicators:

1. higher answer quality under fixed token budgets
2. stable context assembly latency in production

### Track B: Policy and Execution Governance

Focus:

1. deeper planner-native policy contracts
2. stronger decision provenance coverage
3. improved adaptation safety and rollout guardrails

Success indicators:

1. higher decision-link and feedback-link coverage
2. reduced rollout regressions in policy updates

### Track C: Operability and Evidence

Focus:

1. expanded gate automation
2. richer benchmark packs and evidence bundles
3. faster incident replay workflows

Success indicators:

1. shorter mean-time-to-diagnosis in incident drills
2. more complete release evidence packages

### Track D: Ecosystem Integration

Focus:

1. stronger MCP and framework integration patterns
2. better SDK ergonomics for production teams
3. cleaner multi-runtime interoperability guidance

Success indicators:

1. reduced integration time-to-first-value
2. lower integration-specific failure rates

## Planning Horizons

### Near-Term

1. strengthen context orchestration presets and tuning guidance
2. standardize benchmark snapshot reproducibility for release workflows
3. improve operator-facing diagnostics for policy-loop health

### Mid-Term

1. broaden planner-native policy contracts
2. extend replay tooling around URI-linked execution chains
3. mature integration guidance for mixed-runtime agent stacks

### Long-Term

1. strategy-memory capabilities for long-horizon tasks
2. multi-layer context orchestration for complex orchestration graphs
3. enterprise-scale memory operating patterns and controls

## How to Use This Roadmap

1. Treat roadmap tracks as capability direction, not fixed release promises.
2. Prioritize tracks based on your workload constraints (latency, control, compliance).
3. Validate upgrades with production gates and benchmark evidence before rollout.
