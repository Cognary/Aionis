# Role-Based Reading Paths

Use these paths to reach the right depth without reading the whole documentation front to back.

## Local Builder Path

Best for: individual builders, Codex users, local agent workflow testing.

1. [Choose Lite vs Server](/public/en/getting-started/07-choose-lite-vs-server)
2. [5-Minute Onboarding](/public/en/getting-started/02-onboarding-5min)
3. [Lite Operator Notes](/public/en/getting-started/04-lite-operator-notes)
4. [Lite Public Beta Boundary](/public/en/getting-started/05-lite-public-beta-boundary)

Outcome:

1. Start Lite, verify health, understand current beta limits, and know where local feedback belongs.

## Integration Engineer Path

Best for: SDK users, API integrators, MCP/Codex integration work.

1. [Get Started](/public/en/getting-started/01-get-started)
2. [Build Memory Workflows](/public/en/guides/01-build-memory)
3. [API Reference](/public/en/api-reference/00-api-reference)
4. [SDK Guide](/public/en/reference/05-sdk)
5. [Integrations](/public/en/integrations/00-overview)
6. [Codex Local](/public/en/integrations/05-codex-local)

Outcome:

1. Run real write, recall, planning, replay, and Dev MCP flows in a working integration.

## Self-Hosted Platform Path

Best for: teams evaluating Server, self-hosted production, and operational rollout.

1. [Choose Lite vs Server](/public/en/getting-started/07-choose-lite-vs-server)
2. [Operate and Production](/public/en/operate-production/00-operate-production)
3. [Operator Runbook](/public/en/operations/02-operator-runbook)
4. [Production Core Gate](/public/en/operations/03-production-core-gate)
5. [Standalone to HA Runbook](/public/en/operations/06-standalone-to-ha-runbook)

Outcome:

1. Understand what belongs in self-hosted Server, what remains outside the current public surface, and how to operate the production path.

## Runtime Architect Path

Best for: architects, platform designers, and contributors who need the kernel model.

1. [Core Concepts](/public/en/core-concepts/00-core-concepts)
2. [Architecture](/public/en/architecture/01-architecture)
3. [Context Orchestration](/public/en/context-orchestration/00-context-orchestration)
4. [Policy and Execution Loop](/public/en/policy-execution/00-policy-execution-loop)
5. [Benchmarks](/public/en/benchmarks/05-performance-baseline)
6. [Aionis One-Page](/public/en/about/02-aionis-onepage)

Outcome:

1. Understand the runtime model, why replay/context/policy live in one kernel, and where evidence exists for the main product claims.

## Product Reading Rule

1. Start from the product path you need: Lite, integration, or self-hosted Server.
2. Use architecture pages after you know which product path you are evaluating.
3. Treat `about/*` pages as support material, not the first stop.
