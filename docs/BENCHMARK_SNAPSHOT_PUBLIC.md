# Benchmark Snapshot (Public)

Last updated: `2026-03-01`

This page is the external benchmark disclosure for Aionis. It consolidates:

1. performance snapshot (latency + error rate)
2. policy-loop effectiveness snapshot (success/stability/controllability)
3. reproducible commands and artifact locations

## 1) Scope Statement

The snapshot claims only what is measured in reproducible runs:

1. API runtime performance under the documented benchmark profile
2. policy-loop A/B behavior in the documented `XMB-006` scenario
3. governance/evidence gate pass status for the same evidence run

Non-claims:

1. this page does not claim universal performance across all hardware/network topologies
2. this page does not claim unconstrained autonomous learning
3. "self-learning" in Aionis means feedback-driven policy adaptation under governance gates

## 4) Dataset Generation

### 4.1 Performance Dataset

```bash
npm run -s job:perf-seed -- \
  --scope perf \
  --tenant-id default \
  --src-scope default \
  --src-tenant-id default \
  --events 100000 \
  --topics 1000 \
  --reset
```

### 4.2 Policy-Loop Dataset

`XMB-006` is generated by benchmark harness writes in isolated scopes derived from `--scope-prefix`.
No manual seed step is required when running `bench:aionis:v01` or `evidence:weekly`.

## 5) Metric Definitions

### 5.1 Performance Metrics

1. `recall p95/p99`: p95/p99 latency for recall benchmark cases
2. `write p95/p99`: p95/p99 latency for write benchmark cases
3. `failure rate`: failed requests / total requests in benchmark case
4. `SLO result`: pass/fail against configured gate thresholds

### 5.2 Policy-Loop Metrics (`XMB-006`)

1. `baseline success rate`: retrieval-only first-candidate heuristic success ratio
2. `policy success rate`: rules + tools/select + tools/feedback loop success ratio
3. `selection switches`: count of tool-selection flips across repeated episodes
4. `feedback link coverage`: ratio of feedback rows correctly linked to decisions
5. `source rule coverage`: ratio of decisions carrying expected source rule attribution
