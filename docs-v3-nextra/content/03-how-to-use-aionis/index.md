# How To Use Aionis

This layer exists for developers who are ready to try Aionis, not just evaluate the idea.

The goal of this section is simple:

Get a developer from zero to first successful Aionis call in under 30 minutes.

## Success Criteria

Within the first session, a developer should be able to:

1. run Lite locally or point to a Server endpoint
2. complete one `write -> recall_text` loop
3. store and recover one handoff
4. understand where replay and policy fit next

## Integration Priorities

The best first integrations for external users are:

1. Codex local workflows
2. LangGraph-based agent runtimes
3. MCP-based coding-agent setups

## Choose The Lowest-Friction Entry

The best starting point depends on what you need to prove internally:

1. choose [Quickstart](./quickstart) if you need a first success in minutes
2. choose [Complete Flow](./complete-flow) if you need to show continuity as one connected story
3. choose [SDK And HTTP](./sdk-and-http) if integration design is the real question
4. choose [Codex](./codex), [MCP](./mcp), or [LangGraph](./langgraph) if adoption depends on fitting the current stack

Those are the paths where Aionis value becomes obvious quickly.

## Fastest Conversion Path

If you want to decide quickly whether Aionis is worth deeper integration:

1. [Quickstart](./quickstart) for the first `write -> recall_text`
2. [Complete Flow](./complete-flow) for the first full continuity loop
3. [SDK And HTTP](./sdk-and-http) for the actual transport choice
4. [Endpoint Reference](../04-reference-and-operations/endpoints) when you need exact contract details

## What Usually Blocks First Success

The common blockers are not subtle:

1. trying to understand every endpoint before making the first call
2. mixing `live-verified` and `code-backed` surfaces as if they were the same
3. evaluating replay before proving memory and handoff
4. expecting the first session to look like a finished production deployment

This section is intentionally ordered to avoid those mistakes.

## After The First Success

Once the first loop works, the best next step is not more overview reading.

It is usually one of these:

1. move to [Complete Flow](./complete-flow) if you need to show continuity end to end
2. move to [SDK And HTTP](./sdk-and-http) if transport choice matters next
3. move to [Verification Status](../04-reference-and-operations/verification-status) if you need to judge implementation risk
4. move to [Endpoint Reference](../04-reference-and-operations/endpoints) if another team will own the actual integration
