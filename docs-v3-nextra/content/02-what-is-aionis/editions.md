# Editions

Aionis should be explained as one product with multiple adoption paths, not as three unrelated things.

## Lite

Lite is the lowest-friction path into Aionis.

Use it when you want:

1. local single-user workflows
2. SQLite-backed core continuity features
3. fast evaluation for Codex or MCP workflows

Lite should feel like a real evaluation path, not a teaser:

1. it is real
2. it is useful
3. it is intentionally bounded

## Server

Server is the self-hosted production path.

Use it when you want:

1. team ownership
2. admin and control surfaces
3. automation surfaces
4. production operations discipline

Server is the self-hosted production edition of the same core runtime.

## Cloud

Cloud is the managed direction.

The right way to read Cloud here is:

1. managed control-plane direction
2. governance and hosted operations direction
3. not the current public open-repo runtime surface

That keeps the story ambitious without pretending the public repository already ships the entire Cloud product.
